> 创建日期：2026-03-03
> 状态：需求评审阶段
> 基于现有项目：`课程排期表2026.html`

---

## 一、项目背景

现有一个静态 HTML 课程排期表页面，课程数据硬编码在 JS 中。需要升级为可维护、可通知的课程排期系统，实现：
1. 在飞书多维表格中管理课程数据
2. 课表变更和上课提醒通过邮件通知
3. 网页自动从飞书数据源生成并部署

## 二、名词解释

| 术语 | 通俗解释 |
|------|---------|
| **飞书多维表格** | 飞书里的在线表格工具（类似 Excel），支持多人协作编辑，可以通过 API 被程序读取数据。我们用它来录入和管理课程信息。 |
| **GitHub Actions** | GitHub 提供的免费自动化服务。可以设定时任务（如"每天晚上8点"），自动执行我们写好的脚本。类似于一个免费的定时机器人。 |
| **GitHub Pages** | GitHub 提供的免费网页托管服务。把 HTML 文件放上去就能通过网址访问，无需购买服务器。 |
| **163 SMTP** | 网易 163 邮箱的邮件发送接口。SMTP 是邮件发送的标准协议，163 免费提供此服务。我们用一个 163 邮箱作为"发件人"，通过程序自动发送通知邮件给助教和相关人员。**不需要额外付费**，只需一个 163 邮箱账号并开启 SMTP 授权码即可。 |
| **飞书开放平台** | 飞书提供的开发者平台。我们需要在上面创建一个"应用"，获取访问凭证（类似钥匙），这样程序才能读取飞书多维表格中的数据。创建应用免费，操作由技术人员完成。 |
| **Keepalive（保活机制）** | GitHub 会自动禁用 60 天没有活动的定时任务。为防止此情况，我们设一个每周自动运行的小任务来保持仓库活跃。 |

## 六、V1 需求清单

### 模块 1：飞书多维表格（数据源）

#### 1.1 创建飞书多维表格

**描述**：在飞书中创建课程排期专用多维表格，作为系统唯一数据源。

**表结构设计**：

**表 1：课程信息表**

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| 课程名称 | 文本 | 是 | 课程全称 | 省考申论训练营 |
| 课程简称 | 文本 | 是 | 日历上显示的简称 | 申论 |
| 课程代号 | 文本 | 是 | 系统内部标识（英文） | shenlun |
| 颜色 | 文本 | 是 | HEX 色值 | #4A90D9 |
| 排课模式 | 单选 | 是 | 每周X / 每隔N天 / 连续 / 自定义 | 每周X |
| 排课参数 | 文本 | 否 | 模式参数，格式见下方说明 | 二,四 |
| 开始日期 | 日期 | 是 | 课程起始日 | 2026-02-15 |
| 结束日期 | 日期 | 是 | 课程截止日 | 2026-03-12 |
| 自定义日期 | 文本 | 否 | 排课模式为"自定义"时填写 | 2/15,2/18,2/20 |
| 排除日期 | 文本 | 否 | 需要跳过的日期 | 3/1,3/8 |
| 状态 | 单选 | 是 | 进行中 / 已结束 / 暂不安排 | 进行中 |
| 备注 | 文本 | 否 | 补充说明 | 指定日期后切换为每周二四 |

**排课模式 & 参数说明**：
- `每周X`：参数填星期几，多个用逗号分隔。如 `二,四` = 每周二和周四
- `每隔N天`：参数填间隔天数。如 `2` = 每隔2天上一次课（含首日）
- `连续`：参数填总天数。如 `14` = 从开始日期起连续14天
- `自定义`：不看参数，看"自定义日期"字段

**表 2：考试信息表**

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| 考试名称 | 文本 | 是 | | 省考申论考试 |
| 关联课程 | 关联 | 否 | 关联到课程信息表 | 省考申论训练营 |
| 考试日期 | 文本 | 是 | 单日或范围 | 2026-03-14 或 2026-03-06~2026-03-10 |

**表 3：通知配置表**

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| 角色 | 文本 | 是 | 收件人角色描述 | 省考申论助教 |
| 邮箱 | 文本 | 是 | | zhujiao@example.com |
| 关联课程 | 关联 | 否 | 关联到课程信息表（空=接收全部通知） | 省考申论训练营 |
| 通知类型 | 多选 | 是 | 变更通知 / 每日提醒 | 变更通知,每日提醒 |

#### 1.2 创建飞书开放平台应用

**描述**：在飞书开放平台创建自建应用，获取 API 凭证。

**所需权限**：
- `bitable:record:read` — 读取多维表格记录
- （V2 需要 `bitable:record:write` — 写入多维表格）

**产出**：
- `app_id`
- `app_secret`
- 多维表格 URL（含 `app_token` 和 `table_id`）

**操作步骤（待执行时详细指导）**：
1. 访问 https://open.feishu.cn/app 创建自建应用
2. 在"权限管理"中开通多维表格读取权限
3. 在多维表格中添加该应用为协作者
4. 记录 app_id、app_secret、app_token、table_id

### 模块 2：数据拉取 & 解析

#### 2.1 飞书 API 数据拉取脚本

**描述**：Python 脚本，通过飞书 Open API 读取多维表格数据。

**输入**：飞书 API 凭证（环境变量）+ 表格 ID
**输出**：`schedule.json`

**逻辑**：
1. 获取 `tenant_access_token`（通过 app_id + app_secret）
2. 读取课程信息表、考试信息表、通知配置表
3. 根据排课模式 + 参数 + 起止日期 → 计算出每门课的所有上课日期
4. 处理排除日期
5. 输出标准化的 `schedule.json`

#### 2.2 `schedule.json` 数据格式

```json
{
  "generated_at": "2026-03-03T20:00:00+08:00",
  "courses": [
    {
      "id": "shenlun",
      "name": "省考申论训练营",
      "short_name": "申论",
      "color": "#4A90D9",
      "status": "进行中",
      "dates": ["2026-02-15", "2026-02-18", "2026-02-20", "..."],
      "note": ""
    }
  ],
  "exams": [
    {
      "name": "省考申论考试",
      "course_id": "shenlun",
      "dates": ["2026-03-14"]
    }
  ],
  "notifications": [
    {
      "role": "省考申论助教",
      "email": "zhujiao@example.com",
      "course_ids": ["shenlun"],
      "types": ["变更通知", "每日提醒"]
    }
  ]
}
```

### 模块 3：网页生成 & 部署

#### 3.1 HTML 模板化改造

**描述**：将现有 `课程排期表2026.html` 中硬编码的课程数据改为从 `schedule.json` 动态读取。

**改造要点**：
- 删除 JS 中硬编码的课程数据
- 改为 `fetch('schedule.json')` 或构建时注入
- 图例、统计卡片、日历、明细表均从数据动态生成
- "今天"标记改为实时计算（`new Date()`），不再硬编码
- 保持现有 UI 风格不变

#### 3.2 GitHub Actions — 构建 & 部署

**触发条件**：
- 定时：每小时（或自定义频率）检查飞书数据变化
- 手动：workflow_dispatch 支持手动触发

**流程**：
1. 拉取飞书多维表格数据
2. 生成 `schedule.json`
3. 注入到 HTML 模板
4. 部署到 GitHub Pages

### 模块 4：通知功能

#### 4.1 变更检测

**描述**：对比当前 `schedule.json` 与上一次的版本，检测变化。

**检测项**：
- 新增课程
- 删除课程
- 课程日期变化（新增日期、删除日期）
- 考试日期变化

**产出**：变更摘要（用于邮件内容）

**实现方式**：
- 上一次的 `schedule.json` 通过 Git 历史或缓存文件保留
- Python 脚本做 JSON diff

#### 4.2 变更通知邮件

**触发条件**：检测到课表变化
**收件人**：通知配置表中"通知类型"包含"变更通知"的人员
**邮件内容**：
```
主题：【课程排期更新】2026-03-03 课表已变更

您好，课程排期表已更新，变化如下：

[变更摘要]
- 省考申论训练营：3月15日新增一次课
- 广东面试正课：结束日期从3/7改为3/5

查看完整课表：https://xiaomai-cool.github.io/xxx

此邮件由系统自动发送。
```

**技术方案**：
- 163 邮箱 SMTP（复用 future-message 方案）
- 凭证存为 GitHub Actions Secrets

#### 4.3 每日上课提醒

**触发条件**：GitHub Actions cron，每天北京时间 20:00 运行
**逻辑**：
1. 读取 `schedule.json`
2. 检查明天（T+1）有没有课
3. 如果有，查找对应课程的助教邮箱
4. 发送提醒邮件

**邮件内容**：
```
主题：【上课提醒】明天（3月4日 周三）有课

您好，明天有以下课程：

📚 面试长期班 — 3月4日（周三）
   请通知学生按时上课。

查看完整课表：https://xiaomai-cool.github.io/xxx

此邮件由系统自动发送。
```

#### 4.4 Keepalive 机制

**描述**：防止 GitHub 因仓库 60 天无活动自动禁用 Actions cron。
**方案**：复用 future-message 的 keepalive 方案，每周定时提交一次心跳。
